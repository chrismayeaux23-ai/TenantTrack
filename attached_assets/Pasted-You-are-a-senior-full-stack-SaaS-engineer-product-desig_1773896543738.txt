==================================================
PRODUCT VISION
==================================================

VendorTrust is a landlord and property manager platform focused on one painful problem:

Finding, tracking, and trusting vendors for maintenance and repair work.

Landlords constantly deal with:
- unreliable contractors
- bad communication
- no centralized vendor history
- no proof of who did good work
- lost maintenance records
- no easy way to compare vendors
- no accountability on price, speed, quality, or professionalism

VendorTrust should become the system of record for:
1. maintenance requests
2. vendor assignments
3. vendor performance history
4. trusted vendor relationships
5. work order tracking
6. maintenance analytics for landlords

This is NOT a marketplace like Angi or Thumbtack.
It is a landlord-first vendor intelligence and maintenance operations platform.

- property managers with small to mid-sized portfolios

Secondary users:
- maintenance coordinators

==================================================

4. MAINTENANCE REQUESTS / WORK ORDERS
Each request should support:
- title
- description
- category (plumbing, electrical, paint, appliance, HVAC, general, etc.)
- priority
- status
- property
- unit
- tenant name/contact if needed
- photos/files placeholder support if feasible
- created date

- notes/internal comments

Statuses like:
- new
- needs vendor
- assigned
- scheduled
- in progress
- completed
- cancelled

5. VENDOR MANAGEMENT
This is the heart of the app.

Build a dedicated vendor system with:
- vendor profile
- company/person name
- trade/category
- phone/email
- service area
- license/insurance fields
- notes
- tags
- active/inactive status

Vendor detail page should show:
- contact details
- trade type
- jobs completed
- average rating
- on-time percentage
- average response time
- average cost per job
- internal notes
- recent work history

6. VENDOR TRUST SCORE
Create a simple but powerful internal vendor scoring model.

Build a calculated score based on factors such as:
- average rating
- completion rate
- on-time completion
- communication responsiveness
- repeat usage
- number of completed jobs
- issue/complaint count

Display this as:
- trust score badge
- vendor ranking
- sortable vendor list

Make the formula modular in code so I can adjust weights later.

7. ASSIGN VENDORS TO JOBS
- Assign one vendor to a maintenance request
- Show vendor selection dropdown/modal
- Track assignment date
- Track scheduling date
- Show vendor on job card and detail page
- Make reassignment possible

8. JOB COMPLETION + VENDOR REVIEW LOOP
When a job is completed, landlord/property manager can review vendor on:
- quality
- speed
- communication
- professionalism
- cost fairness
- overall rating
- private notes

These reviews must feed the Vendor Trust Score.

9. VENDOR WORK HISTORY
Each vendor should have a job history timeline including:
- job title
- property
- category
- dates
- status
- cost if available
- review outcome
- notes

10. SEARCH, FILTERS, SORTING
Support across main views:
- filter requests by status, priority, property, category
- filter vendors by trade, score, status, tags
- sort vendors by trust score, recent activity, rating, job count
- search by vendor name, property, request title, unit

11. ACTIVITY FEED
Track important events:
- request created
- vendor assigned
- status changed
- request completed
- vendor reviewed
- property added

Show a useful landlord-facing activity feed on dashboard.

12. BASIC ANALYTICS
Create a lightweight analytics section showing:
- average completion time
- most common maintenance categories
- vendor leaderboard
- open vs completed requests
- overdue requests
- average vendor rating
- maintenance volume by property
